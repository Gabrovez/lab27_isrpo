### Главные выводы

1. **Minimal API позволяет быстро создавать легковесные веб-сервисы без избыточной структуры контроллеров и представлений.** В отличие от MVC, здесь вся логика сосредоточена в одном файле `Program.cs`, что упрощает разработку небольших API и микросервисов.

2. **Порядок регистрации middleware критически важен.** Если зарегистрировать маршрут (`app.MapGet(...)`) до middleware (`app.Use(...)`), то middleware не будет выполнен для этого маршрута, так как запрос уже будет обработан и ответ отправлен клиенту. Middleware работает как конвейер: каждый элемент должен быть вызван до того, как запрос достигнет конечного обработчика.

3. **ASP.NET Core автоматически преобразует имена свойств из PascalCase в camelCase при сериализации в JSON.** Это сделано для соответствия стандартам JavaScript/JSON, где принято использовать camelCase. Такое поведение можно изменить через настройки сериализатора, но по умолчанию оно обеспечивает совместимость с фронтенд-приложениями.


| Характеристика             | ASP.NET Core                                      |
|---------------------------|--------------------------------------------------|
| **Создание сервера**           | `WebApplication.CreateBuilder()`                 |
| **Запуск**                     | `app.Run()`                                      |
| **Маршрут GET**                | `app.MapGet("/", fn)`                            |
| **Параметр маршрута**          | `{name} → (string name)`                         |
| **Возврат JSON**               | `return new {...} / Results.Ok(...)`            |
| **Middleware**                 | `app.Use(async (ctx, next) => ...)`              |

| **Статус ответа**              | `Results.NotFound(...)`                          |
| **Тип данных**                 | Строгие (C# record, class)                       |

### Ответы на вопросы

#### 1. Чем Minimal API отличается от MVC в ASP.NET Core?

Minimal API — это упрощённый подход к созданию веб-приложений, где вся логика размещается в одном файле `Program.cs`. Он не требует создания отдельных контроллеров, моделей и представлений. Подходит для небольших API, микросервисов или прототипов.

MVC (Model-View-Controller) — это архитектурный паттерн, предполагающий разделение ответственности: модель хранит данные, представление отображает интерфейс, контроллер обрабатывает запросы. Требует более сложной структуры проекта, но удобен для больших приложений с UI и бизнес-логикой.

#### 2. Что произойдёт, если поставить `app.MapGet(...)` до `app.Use(...)`?

Если маршрут зарегистрирован до middleware, то middleware **не будет выполнен** для этого маршрута. Запрос сразу попадёт в обработчик маршрута, минуя всю цепочку middleware. Это может привести к отсутствию логирования, авторизации, проверки заголовков и других важных функций, которые должны выполняться до обработки запроса.

#### 3. Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON?

Это сделано для соблюдения соглашений именования в JavaScript и JSON, где переменные и свойства объектов обычно пишутся в camelCase (например, `firstName`, `lastName`). Поскольку большинство клиентов (браузеры, мобильные приложения) ожидают именно такой формат, ASP.NET Core по умолчанию применяет эту трансформацию. Поведение можно изменить через настройки `JsonSerializerOptions.PropertyNamingPolicy`.

#### 4. Что означает код ответа HTTP 401? А 404? А 200?

- **200 OK** — запрос успешно обработан, сервер вернул ожидаемый результат.
- **401 Unauthorized** — клиент не авторизован. Требуется аутентификация (например, передача токена или ключа).
- **404 Not Found** — запрошенный ресурс не найден на сервере. Возможно, неверный URL или маршрут не зарегистрирован.

#### 5. Чем `dotnet run` отличается от `dotnet watch run`?

- `dotnet run` — запускает приложение один раз. При изменении кода нужно вручную останавливать сервер (Ctrl+C) и запускать снова.
- `dotnet watch run` — запускает приложение в режиме горячей перезагрузки. При любом изменении исходных файлов сервер автоматически перезапускается, что ускоряет процесс разработки.
